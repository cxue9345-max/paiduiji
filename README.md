## 5. myjs 基础配置（基础功能）

> 来自“myjs 基本配置填写方法(基础功能)”文档要点。

### 5.1 插件管理设置

- 把“写一下你的名字”改成你的 B 站 ID。
- 说明：`2024-04-29` 版本及以后通常无需调整。

### 5.2 黑名单成员

- 黑名单含义：对方可在直播间说话，但不能参与排队。
- 重要限制：
  - 由于插件缓存机制，**必须在开播前**添加黑名单。
  - 如果开播后修改黑名单，可能导致当前排队信息被清空。

### 5.3 舰长插队（预设中）

- 当前说明为“预设中，未实装”。
- 可先了解：
  - 若舰长在职，通常无需填写。
  - 该配置用于主播自定义“在职/过期舰长插队权限”策略。

## 6. myjs 进阶功能（可选）

> 原文提示：**“能用再试试新功能”**。基础功能稳定后再配置。

### 6.1 云湖通知器（可选）

- 不使用可留空。
- 作用：排队列表每次更新会触发一次推送。
- 云湖开放文档：
  - https://www.yhchat.com/document/400-410

示例变量：

```js
var YHbotId = ""; // 群号或个人ID；群需先申请机器人并拉群
var YHbot_msg_type = ""; // 推送到群填 group，个人填 user
var YHbot_webhook_token = ""; // 机器人的 webhook token
```

说明：webhook 形式示例（原文）
- `https://chat-go.jwzhd.com/open-apis/v1/bot/send?token=xxxxx`

### 6.2 企业微信通知器（可选）

- 填 webhook 完整地址。
- 原文备注：该功能“好像坏了”。

```js
var WX_webhook = ""; // https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxx
```

## 9. 自建后端（backend）运行说明

当前版本为自建后端。

### 9.1 Windows 运行

- 直接运行 `run.bat`。
- 记得允许网络权限。

### 9.2 其他方式运行

- 解压后在当前目录打开控制台，执行：

```bash
go run ./services/backend
```

- 默认端口：WebSocket `23333`、配置 API `23334`。
- 如需改端口，请修改 `services/backend/main.go` 中 `localWSAddr` / `localAPIAddr`。

### 9.3 配置与退出

- 浏览器打开：`http://127.0.0.1:23333`
- 退出方式：直接关闭命令行窗口。

### 9.4 更新与反馈

- 后端逻辑参考：https://github.com/BanqiJane/Bilibili_Danmuji/
- 当前仓库使用自建 Go 后端，不再依赖 danmuji-green 绿色包。

# 指令总表（排队姬）

> 按你的要求，整理为「指令 - 指令用法 - 权限人 - 备注」表格。

| 指令 | 指令用法 | 权限人 | 备注 |
|---|---|---|---|
| 排队 | 直接发送：`排队` | 所有人 | 将发送者加入队列末尾。 |
| 排队（带内容） | 发送：`排队 + 空格 + 内容` | 所有人 | 例如：`排队 原神`，把内容一起写入队列。 |
| 超级排 | 发送：`超级排 + 空格 + 内容` | 所有人 | 与排队类似，展示样式更醒目。 |
| 取消排队 | 发送：`取消排队` 或 `排队取消` | 所有人 | 取消自己当前排队。 |
| 舰长插队 | 发送：`插队` | 舰长 | 仅在管理员开启“舰长插队”后生效。 |
| 指定位置插队 | 发送：`插队 <位置> <内容>` | 管理员 | 例如：`插队 2 胡桃`，插入第 2 位。 |
| 无影插队 | 发送：`无影插 <位置> <内容>` | 管理员 | 与插队类似，内部处理方式不同。 |
| 删除队列项 | 发送：`del<数字>` / `<数字>del` / `删除<数字>` / `完成<数字>` | 管理员 | 例如：`del1` 删除第 1 位。 |
| 添加队列项 | 发送：`add <内容>` / `新增 <内容>` / `添加 <内容>` | 管理员 | 直接向队尾新增一条内容。 |
| 拉黑用户 | 发送：`拉黑 <用户名>` | 管理员 | 被拉黑用户本场不可再次排队。 |
| 取消拉黑 | 发送：`取消拉黑 <用户名>` | 管理员 | 解除本场黑名单。 |
| 添加临时管理员 | 发送：`添加管理员 <用户名>` | 管理员 | 直播进行中添加本场临时管理。 |
| 取消临时管理员 | 发送：`取消管理员 <用户名>` | 管理员 | 取消本场临时管理权限。 |
| 开启舰长插队 | 发送：`开启舰长插队` | 管理员 | 打开舰长发送“插队”的能力。 |
| 关闭舰长插队 | 发送：`关闭舰长插队` | 管理员 | 关闭舰长插队能力。 |
| 暂停排队 | 发送：`暂停排队功能` 或 `关闭自助排队` | 管理员 | 暂停观众自助排队。 |
| 恢复排队 | 发送：`恢复排队功能` 或 `恢复自助排队` | 管理员 | 恢复观众自助排队。 |
| 允许房管继承管理 | 发送：`允许房管成为插件管理员` | 管理员 | 允许房管拥有插件管理员权限。 |
| 停止房管继承管理 | 发送：`停止房管成为插件管理员` | 管理员 | 关闭房管继承管理员权限。 |
| 开启仅粉丝排队 | 发送：`开启仅粉丝可排队` | 管理员 | 限制为粉丝可排队。 |
| 关闭仅粉丝排队 | 发送：`关闭仅粉丝可排队` | 管理员 | 取消粉丝限制。 |
| 设置被插队次数 | 发送：`设置被插队次数 <数字>` | 管理员 | 设置普通用户可被舰长插队次数。 |
| 设置排队人数上限 | 发送：`设置排队人数 <数字>` / `设置排队上限 <数字>` / `设置排队人数上限 <数字>` | 管理员 | 动态修改队列最大人数。 |
| 设置确认时间 | 发送：`设置确认时间 <数字>` | 管理员 | 叫号确认等待时间（秒）。 |
